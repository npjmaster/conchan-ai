# AGENTS.md

## 1. 目的

本ファイルは、AIエージェント（Codexなど）が本プロジェクトを安全かつ一貫性を保って開発するための指示書である。

本プロジェクトは「献ちゃんAI」という献立自動生成Webアプリである。

---

## 2. 基本方針

- 既存構造を大きく壊さない
- 最小変更で修正する
- 不明点は勝手に仕様変更せず、安全側に倒す
- 常に「動くこと」を優先する

---

## 3. 最重要ルール（必ず守る）

### 3.1 AI出力

- AIの出力は必ずJSONのみとする
- JSON.parseは必ずtry-catchで囲む
- 不正JSONの場合は1回だけ再生成する
- それでも失敗した場合はエラー表示する

---

### 3.2 セキュリティ

- APIキーをフロントエンドに出さない
- .envはGitに含めない
- ユーザーごとのデータは必ず分離する
- 他ユーザーのデータは取得できないようにする

---

### 3.3 データ整合性

- 献立と買い物リストは必ず一致させる
- 献立に存在しない食材を追加しない
- 主菜の主要食材は必ず含める

NG例：
- さばの生姜煮 → さばが無い
- 献立に無い味噌が追加される

---

### 3.4 日付処理

- 入力された開始日をそのまま使用する
- タイムゾーンはJST固定
- 日付がズレることは禁止

---

### 3.5 食事順序

表示順は固定：

1. 朝食
2. 昼食
3. 夕食

AIの出力順には依存しない

---

## 4. 実装ルール

### 4.1 TypeScript

- 型エラーを残さない
- anyは極力使わない

---

### 4.2 Prisma / DB

- Prismaを使用する
- raw SQLは禁止
- ユーザーIDで必ずフィルタする

---

### 4.3 API

- 例外は必ずcatchする
- エラー時はJSONで返す

```json
{ "error": "エラーメッセージ" }
```

---

### 4.4 UI

- スマホ優先
- シンプル・あたたかみ
- ボタンは押しやすくする

---

## 5. エラーハンドリング

### 5.1 OpenAI未設定

- 「AI機能が未設定です」と表示
- モックは使わない

---

### 5.2 JSONパース失敗

- 再生成1回
- 失敗したらエラー表示

---

### 5.3 DBエラー

- クラッシュさせない
- 「現在利用できません」と表示

---

## 6. コマンド

作業後は必ず実行：

```bash
npm run build
npx prisma generate
```

---

## 7. 優先順位

1. 既存機能を壊さない
2. バグ修正
3. データ整合性
4. UI改善
5. 新機能追加

---

## 8. MVPの完了条件

以下をすべて満たすこと：

- AIが実際に動作する
- 日付が正しい
- 献立と買い物リストが一致する
- エラーで画面が落ちない

---

## 9. 禁止事項

- 大規模リファクタリング
- 無断でDB構造変更
- モックで誤魔化す
- 不要なライブラリ追加

---

## 10. 不明点がある場合

- シンプルな実装を選択する
- 動作することを優先する
